- Spam or off-topic discussions
- Deliberately disrupting conversations

   - Implement measures to prevent recurrence

## üéì Conflict Resolution

### Disagreements

Disagreements about code or design are normal and healthy!

**When you disagree:**

1. **Stay calm and respectful**
   - Assume good intent
   - Use "I" statements
   - Avoid blame language

2. **Listen actively**
   - Try to understand their perspective
   - Ask clarifying questions
   - Acknowledge valid points

3. **Find common ground**
   - Identify shared goals
   - Look for compromise solutions
   - Consider alternatives

4. **Escalate if needed**
   - Bring in a maintainer for complex issues
   - Document the discussion
   - Request neutral mediation

**Example Conversation:**

```
‚ùå Bad:
"Your code is inefficient and shows you don't understand databases."

‚úÖ Good:
"I have a concern about the query performance here. 
Have you considered using an index? Here's an example..."
```
